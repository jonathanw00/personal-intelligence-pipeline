## 1. Project overview

**What it does:** Captures articles and YouTube videos via Telegram bot, processes them with Claude Haiku, and deposits structured markdown notes into Obsidian vault on Synology DS1522 NAS.

**Guiding principles:**

- Config-first: anything tuneable lives in `config.yaml`, not in code
- Adapter pattern: each content type is an isolated adapter; adding new types touches nothing else
- Mirror voice pipeline conventions: same `.env`, lock file, log rotation, `processing/done/failed/` pattern
- Never write to `Contacts/` or any CRM-related folders
- EspoCRM remains air-gapped â€” zero pipeline connection, ever

## 5. Frontmatter schema

Every note written by the pipeline uses this frontmatter:

```yaml
---
created: 19-Apr-2026
source: https://original-url-here.com
type: article        # options: article | youtube | kindle
tags:
  - tag-one
  - tag-two
status: inbox
summary: "1â€“3 sentence summary of the core idea, argument, or key takeaways."
daily-note: "[[19-Apr-2026]]"
---
```

**Rules:**

- `type` is always one of the defined options â€” new adapters add to this list
- `status` is always `inbox` on creation â€” the human promotes it from there
- `tags` use `kebab-case`, no hashtags, 4â€“6 per note
- `daily-note` is a wikilink to the journal daily note for that date
- Never write `client:` or any contact-identifying fields in pipeline-generated notes


```markdown

1â€“3 sentences. Quotes used here are ==highlighted== in the source text below.
```

- For content above `full_text_threshold` (set in `config.yaml`), capture context windows around quotes rather than full text
- Context window size is tuneable via `config.yaml`

## 6. Job file format

The Telegram bot accepts two input types and writes a `.job` file to `/inbox/` for each.

**URL input** (standard path):

```json
{
  "input_type": "url",
  "url": "https://example.com/article",
  "type": "article",
  "received_at": "2026-04-08T14:32:00",
  "chat_id": "123456789"
}
```

**Pasted text input** (for paywalled articles, PDFs opened in browser, etc.):

```json
{
  "input_type": "text",
  "url": null,
  "type": "article",
  "text": "Full article text pasted here...",
  "received_at": "2026-04-08T14:32:00",
  "chat_id": "123456789"
}
```

**Bot detection logic:**

- Message starts with `http://` or `https://` â†’ URL path â†’ router detects type, adapter fetches content
- Message is long-form text (above `min_paste_words` threshold in `config.yaml`) â†’ text path â†’ skip extraction, pass directly to Haiku
- Message is short text (below threshold) â†’ bot replies asking for clarification

**Recommended workflow for paywalled content:** Use the [Markdownload](https://github.com/deathau/markdownload) browser extension to copy full article text, then paste into Telegram. Works for any site regardless of subscription or paywall.

## 8. Adapters

### Article adapter (`adapters/article.py`)

- Library: `trafilatura`
- Input: URL string
- Output: `{"text": "...", "title": "...", "author": "...", "date": "..."}`
- Fallback: `readability-lxml` if trafilatura returns empty

### YouTube adapter (`adapters/youtube.py`)

- Library: `yt-dlp`
- Input: YouTube URL
- Output: `{"text": "...", "title": "...", "channel": "...", "date": "..."}`
- Method: fetch auto-generated captions as plaintext
- Fallback: log error and move to `failed/` if no captions available
